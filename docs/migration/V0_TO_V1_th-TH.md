# คู่มือการย้ายระบบ (จาก v0 เป็น v1)

หากคุณกำลังใช้ Bedrock Claude Chat เวอร์ชันก่อนหน้านี้ (~`0.4.x`) คุณจำเป็นต้องปฏิบัติตามขั้นตอนด้านล่างเพื่อย้ายระบบ

## ทำไมฉันต้องทำ?

การอัปเดตครั้งใหญ่นี้มีการอัปเดตความปลอดภัยที่สำคัญ

- การจัดเก็บฐานข้อมูลเวกเตอร์ (เช่น pgvector บน Aurora PostgreSQL) ตอนนี้ได้รับการเข้ารหัสแล้ว ซึ่งจะทำให้เกิดการแทนที่เมื่อทำการปรับใช้ ซึ่งหมายความว่ารายการเวกเตอร์ที่มีอยู่จะถูกลบ
- เราได้แนะนำกลุ่มผู้ใช้ Cognito `CreatingBotAllowed` เพื่อจำกัดผู้ใช้ที่สามารถสร้างบอต ผู้ใช้ที่มีอยู่ในปัจจุบันจะไม่อยู่ในกลุ่มนี้ ดังนั้นคุณจำเป็นต้องแนบสิทธิ์ด้วยตนเองหากต้องการให้พวกเขามีความสามารถในการสร้างบอต ดู: [การปรับแต่งบอต](../../README.md#bot-personalization)

## ข้อกำหนดเบื้องต้น

อ่าน [คู่มือการย้ายฐานข้อมูล](./DATABASE_MIGRATION_th-TH.md) และกำหนดวิธีการกู้คืนรายการ

## ขั้นตอน

### การย้ายคลังเวกเตอร์

- เปิดเทอร์มินัลของคุณและไปที่ไดเรกทอรีโปรเจกต์
- ดึงแขนงที่คุณต้องการปรับใช้ ทำตามขั้นตอนไปยังแขนงที่ต้องการ (ในกรณีนี้คือ `v1`) และดึงการเปลี่ยนแปลงล่าสุด:

```sh
git fetch
git checkout v1
git pull origin v1
```

- หากคุณต้องการกู้คืนรายการด้วย DMS โปรดปิดการหมุนรหัสผ่านและจดรหัสผ่านเพื่อเข้าถึงฐานข้อมูล หากกู้คืนด้วยสคริปต์การย้าย([migrate.py](./migrate.py)) คุณไม่จำเป็นต้องจดรหัสผ่าน
- ลบ[API ที่เผยแพร่](../PUBLISH_API_th-TH.md) ทั้งหมดเพื่อให้ CloudFormation สามารถลบคลัสเตอร์ Aurora ที่มีอยู่
- เรียกใช้ [npx cdk deploy](../README.md#deploy-using-cdk) เพื่อเปลี่ยนแทนคลัสเตอร์ Aurora และ ลบรายการเวกเตอร์ทั้งหมด
- ทำตาม[คู่มือการย้ายฐานข้อมูล](./DATABASE_MIGRATION_th-TH.md) เพื่อกู้คืนรายการเวกเตอร์
- ตรวจสอบว่าผู้ใช้สามารถใช้บอทที่มีอยู่ที่มีความรู้ เช่น บอท RAG

### เพิ่มสิทธิ์ CreatingBotAllowed

- หลังจากการปรับใช้ ผู้ใช้ทั้งหมดจะไม่สามารถสร้างบอทใหม่ได้
- หากคุณต้องการให้ผู้ใช้เฉพาะรายสามารถสร้างบอทได้ ให้เพิ่มผู้ใช้เหล่านั้นในกลุ่ม `CreatingBotAllowed` โดยใช้คอนโซลการจัดการหรือ CLI
- ตรวจสอบว่าผู้ใช้สามารถสร้างบอทได้ โปรดทราบว่าผู้ใช้ต้องเข้าสู่ระบบใหม่
