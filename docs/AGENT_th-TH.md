# เอเจนต์ที่ขับเคลื่อนด้วย LLM (ReAct)

## Agent (ReAct) คืออะไร

Agent เป็นระบบปัญญาประดิษฐ์ขั้นสูงที่ใช้โมเดลภาษาขนาดใหญ่ (LLMs) เป็นเครื่องยนต์การคำนวณหลัก โดยรวมความสามารถในการให้เหตุผลของ LLMs เข้ากับฟังก์ชันเพิ่มเติม เช่น การวางแผนและการใช้เครื่องมือ เพื่อทำงานที่ซับซ้อนด้วยตนเองโดยอัตโนมัติ Agent สามารถแตกประเด็นคำถามที่ซับซ้อน สร้างโซลูชันทีละขั้นตอน และโต้ตอบกับเครื่องมือหรือ API ภายนอกเพื่อรวบรวมข้อมูลหรือดำเนินการงานย่อย

ตัวอย่างนี้ใช้ Agent โดยใช้แนวทาง [ReAct (Reasoning + Acting)](https://www.promptingguide.ai/techniques/react) ReAct ช่วยให้ agent สามารถแก้ปัญหาที่ซับซ้อนโดยการรวมการให้เหตุผลและการกระทำในวงรอบป้อนกลับแบบวนซ้ำ Agent จะดำเนินการผ่านสามขั้นตอนหลัก: การคิด การกระทำ และการสังเกต โดยวิเคราะห์สถานการณ์ปัจจุบันโดยใช้ LLM ตัดสินใจเลือกการกระทำถัดไป ดำเนินการกระทำโดยใช้เครื่องมือหรือ API ที่มี และเรียนรู้จากผลที่สังเกตได้ กระบวนการต่อเนื่องนี้ช่วยให้ agent สามารถปรับตัวเข้ากับสภาพแวดล้อมที่เปลี่ยนแปลง ปรับปรุงความแม่นยำในการแก้ปัญหา และให้โซลูชันที่คำนึงถึงบริบท

## กรณีใช้งานตัวอย่าง

เอเจนต์ที่ใช้ ReAct สามารถนำไปประยุกต์ใช้ในสถานการณ์ต่าง ๆ โดยให้โซลูชันที่แม่นยำและมีประสิทธิภาพ

### การแปลงข้อความเป็น SQL

ผู้ใช้ขอ "ยอดขายรวมของไตรมาสที่ผ่านมา" เอเจนต์จะตีความคำร้องขอนี้ แปลงเป็นคิวรี SQL ดำเนินการกับฐานข้อมูล และนำเสนอผลลัพธ์

### การพยากรณ์ทางการเงิน

นักวิเคราะห์ทางการเงินต้องการพยากรณ์รายได้ของไตรมาสถัดไป เอเจนต์จะรวบรวมข้อมูลที่เกี่ยวข้อง คำนวณโดยใช้แบบจำลองทางการเงิน และสร้างรายงานพยากรณ์อย่างละเอียด โดยสร้างความมั่นใจในความถูกต้องของการคาดการณ์

## การใช้คุณลักษณะ Agent

เพื่อเปิดใช้งานฟังก์ชัน Agent สำหรับแชทบอทที่กำหนดเองของคุณ ให้ทำตามขั้นตอนเหล่านี้:

1. ไปที่ส่วน Agent ในหน้าจอแชทบอทที่กำหนดเอง

2. ในส่วน Agent คุณจะพบรายการเครื่องมือที่สามารถใช้งานได้โดย Agent ตามค่าเริ่มต้น เครื่องมือทั้งหมดจะถูกปิดใช้งาน

3. หากต้องการเปิดใช้งานเครื่องมือ ให้กดสวิตช์ข้างเครื่องมือที่ต้องการ เมื่อเปิดใช้งานเครื่องมือแล้ว Agent จะสามารถเข้าถึงและใช้งานเครื่องมือนั้นได้เมื่อประมวลผลคำถามของผู้ใช้

![](./imgs/agent_tools.png)

> [!สำคัญ]
> สำคัญที่ต้องทราบว่าการเปิดใช้งานเครื่องมือใดๆ ในส่วน Agent จะถือว่าฟังก์ชัน "ความรู้" เป็นเครื่องมือด้วย ซึ่งหมายความว่า LLM จะกำหนดโดยอัตโนมัติว่าจะใช้ "ความรู้" เพื่อตอบคำถามของผู้ใช้ โดยพิจารณาเป็นหนึ่งในเครื่องมือที่มีอยู่

4. ตามค่าเริ่มต้น เครื่องมือ "ค้นหาอินเทอร์เน็ต" จะถูกจัดเตรียมไว้ เครื่องมือนี้ช่วยให้ Agent สามารถดึงข้อมูลจากอินเทอร์เน็ตเพื่อตอบคำถามของผู้ใช้

5. คุณสามารถพัฒนาและเพิ่มเครื่องมือแบบกำหนดเองเพื่อขยายความสามารถของ Agent ดูรายละเอียดเพิ่มเติมเกี่ยวกับการสร้างและบูรณาการเครื่องมือแบบกำหนดเองได้ที่ส่วน [วิธีพัฒนาเครื่องมือของคุณเอง](#how-to-develop-your-own-tools)

## วิธีพัฒนาเครื่องมือของคุณเอง

เพื่อพัฒนาเครื่องมือที่กำหนดเองสำหรับ Agent ให้ปฏิบัติตามแนวทางเหล่านี้:

- สร้างคลาสใหม่ที่สืบทอดจากคลาส `AgentTool` แม้ว่าอินเทอร์เฟซจะเข้ากันได้กับ LangChain แต่การใช้งานตัวอย่างนี้มี `AgentTool` คลาสของตนเอง ซึ่งคุณควรสืบทอดจาก ([แหล่งที่มา](../backend/app/agents/tools/agent_tool.py))

- ดูตัวอย่างการใช้งานของ[เครื่องมือคำนวณ BMI](../examples/agents/tools/bmi/bmi.py) ตัวอย่างนี้แสดงวิธีสร้างเครื่องมือที่คำนวณดัชนีมวลกาย (BMI) ตามข้อมูลที่ผู้ใช้ป้อน

  - ชื่อและคำอธิบายที่ประกาศบนเครื่องมือจะถูกใช้เมื่อ LLM พิจารณาว่าควรใช้เครื่องมือใดเพื่อตอบคำถามของผู้ใช้ กล่าวอีกนัยหนึ่ง พวกเขาจะถูกฝังในพรอมพ์เมื่อเรียก LLM ดังนั้นขอแนะนำให้อธิบายอย่างชัดเจนมากที่สุด

- [เลือกได้] เมื่อคุณได้ใช้งานเครื่องมือที่กำหนดเองแล้ว ขอแนะนำให้ตรวจสอบฟังก์ชันการทำงานโดยใช้สคริปต์ทดสอบ ([ตัวอย่าง](../examples/agents/tools/bmi/test_bmi.py)) สคริปต์นี้จะช่วยให้คุณมั่นใจว่าเครื่องมือของคุณทำงานได้ตามที่คาดหวัง

- หลังจากการพัฒนาและทดสอบเครื่องมือที่กำหนดเองเสร็จสิ้น ให้ย้ายไฟล์การใช้งานไปยังไดเรกทอรี [backend/app/agents/tools/](../backend/app/agents/tools/) จากนั้นเปิด [backend/app/agents/utils.py](../backend/app/agents/utils.py) และแก้ไข `get_available_tools` เพื่อให้ผู้ใช้สามารถเลือกเครื่องมือที่พัฒนาได้

- [เลือกได้] เพิ่มชื่อและคำอธิบายที่ชัดเจนสำหรับส่วนหน้า ขั้นตอนนี้เป็นทางเลือก แต่หากคุณไม่ทำขั้นตอนนี้ ชื่อและคำอธิบายเครื่องมือที่ประกาศในเครื่องมือของคุณจะถูกใช้ พวกเขาสำหรับ LLM แต่ไม่ใช่สำหรับผู้ใช้ ดังนั้นขอแนะนำให้เพิ่มคำอธิบายเฉพาะเพื่อปรับปรุง UX

  - แก้ไขไฟล์ i18n เปิด [en/index.ts](../frontend/src/i18n/en/index.ts) และเพิ่ม `name` และ `description` ของคุณใน `agent.tools`
  - แก้ไข `xx/index.ts` เช่นกัน โดยที่ `xx` แทนรหัสประเทศที่คุณต้องการ

- เรียกใช้ `npx cdk deploy` เพื่อปรับใช้การเปลี่ยนแปลงของคุณ นี่จะทำให้เครื่องมือที่กำหนดเองของคุณพร้อมใช้งานในหน้าจอบอทที่กำหนดเอง
